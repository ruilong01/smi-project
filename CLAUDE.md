# Global Maritime Research Intelligence Map

Internal Maritime R&D Intelligence Platform for SMI staff: explore global
maritime research on an interactive globe, view projects by country,
institution, topic and stage, with AI analysis added AFTER extraction only.

## Environment

- Developer machine: **Windows**. Always use `npm.cmd` in commands, not `npm`.
- Target server: AWS Lightsail, Ubuntu Linux, 4GB RAM / 2 vCPU.
- Frontend: React 18 + Vite 6. Data: static JSON generated by
  `scripts/ingestion/` (Node .mjs). No backend exists yet (planned Phase 2).

## Commands

- Dev server:        `npm.cmd run dev`
- Tests:             `npm.cmd test -- --run`
- Production build:  `npm.cmd run build`
- Data sync (once):  `npm.cmd run sync:data`
- Data sync (watch): `npm.cmd run sync:watch`

## Architecture rules (non-negotiable)

1. Do NOT use AI to fetch webpages. Fetching uses public APIs, RSS,
   requests/BeautifulSoup/Scrapy, Playwright only when required.
2. AI is used only AFTER extraction: classification, summarisation,
   relevance, comparison, trends, digest generation.
3. Every real data record must include a source URL and fetched timestamp.
4. Do not expose API keys in frontend code. Use environment variables.
5. Never read or write `.env*` files or any secrets.
6. One failed source must never stop the full pipeline.
7. Cache AI results; only reanalyse on content change, explicit request,
   or prompt/model version change.

## Globe rules

- Keep the 50m atlas (decision confirmed). Performance comes from
  back-face culling + memoised path caching, NOT lower resolution.
- One requestAnimationFrame loop only. Refs for rotation/zoom.
- No React state updates at 60fps. No rebuilding geometry per frame.
- Auto-rotation pauses on hover/drag/popup/panel; resumes after ~1.5s.
- Blue/cyan/teal intensity palette; red = selected focus only.
- Full-screen fixed viewport; no page-level scrolling on the dashboard.

## Working rules

- Inspect existing code before changing it. No blind rewrites.
- Small, reviewable changes. Preserve working features.
- Run tests and production build after changes.
- Never edit node_modules, dist, or package-lock unnecessarily.
- Keep mock data working until real data replaces it; mock data stays
  labelled "Prototype demo data - source verification pending."
- At end of every task report: changes, files modified, test commands,
  how verified, remaining problems, recommended next task.
